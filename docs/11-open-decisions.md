# Decisiones pendientes

## Alta prioridad

- Cantidad de planes y diferencias comerciales.
- Metodo de cobro inicial: manual o integrado.
- Politica ante `past_due`: gracia o bloqueo inmediato.
- Costo base para valuacion: ultimo costo ya esta propuesto para MVP, pero debe confirmarse.

## Media prioridad

- Notificaciones por email para aprobacion, bloqueo y vencimiento.
- Politica exacta de acceso para `approved_pending_payment` y `cancelled`.
- Alcance futuro de exportacion despues de JSON/CSV: por modulo, PDF o formatos contables.

## Baja prioridad

- Migracion automatica desde el prototipo local anterior.
- Reportes PDF fuera del MVP inicial.

## Decision por defecto recomendada

Si una decision no esta cerrada y no bloquea integridad, elegir la opcion mas simple, segura y reversible, y documentarla en este archivo.
